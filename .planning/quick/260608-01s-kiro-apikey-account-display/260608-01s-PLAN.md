# Quick Task 260608-01s: Repair kiro account display for api-key accounts

## Problem

The previous commit (`1d28d9f feat(kiro): show account details on Quota Tracker tab`)
enriched the Kiro connection cards (auth method, region, status, profileArn) but
missed the actual goal: **kiro accounts added via api-key were not being displayed**.

Two concrete symptoms, kiro provider only:

1. **Quota Tracker (dashboard/quota)** — kiro accounts with `authType: "apikey"`
   never appear. Root cause: `USAGE_APIKEY_PROVIDERS` did not include `kiro`, so
   `isUsageEligible()` in `src/app/api/providers/client/route.js` filtered them out
   (and the usage route `src/app/api/usage/[connectionId]/route.js` rejected them too).

2. **Providers tab count mismatch** — the kiro card shows 3 accounts, but clicking
   into the provider detail page shows 4. Root cause: the providers list card counts
   only `authType === "oauth"` connections (`getProviderStats(key, "oauth")`), while
   the detail page filters by `provider` alone (no authType). The 1 api-key kiro
   account is the difference.

## Tasks

### Task 1 — Make api-key kiro eligible for the Quota Tracker
- files: `src/shared/constants/providers.js`
- action: Add `"kiro"` to `USAGE_APIKEY_PROVIDERS`.
- verify: `isUsageEligible` (client/route.js) and the usage route accept api-key kiro.
- done: api-key kiro connections appear in dashboard/quota cards.

### Task 2 — Fix provider card count to include api-key kiro
- files: `src/app/(dashboard)/dashboard/providers/page.js`
- action: Allow `getProviderStats`/`handleToggleProvider` to accept an array of
  authTypes; render the kiro free card with `["oauth", "apikey"]` so the count and
  toggle cover both connection kinds. Other free providers keep `"oauth"`.
- verify: kiro card count matches the detail page (4 vs 4).
- done: no count mismatch; provider toggle affects both oauth and api-key kiro accounts.

## Scope guard
Kiro provider only. No behavior change for other providers (other free providers
still pass the scalar `"oauth"`; `getProviderStats` accepts both scalar and array).
